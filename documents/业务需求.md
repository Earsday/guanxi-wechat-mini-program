## 1. 项目概述

### 1.1 项目背景
现代人的社交网络日益复杂，涉及亲戚、好友、同事、同学、邻里等多种关系类型。这些关系往往具有不同的特点和管理需求，且关系状态可能随时间变化（断联、重联等）。本项目旨在开发一款微信小程序，帮助用户系统化地记录、管理和梳理人际关系网。

**核心设计理念**：本程序维护的关系网络并不一定要从当前用户出发，可以从任意一个人物开始记录和梳理，构建以该人物为中心发散的关系网。这意味着本程序不仅可以管理个人的人际关系，还可以用于梳理小说、电影、历史事件中的人物关系。

### 1.2 项目目标
- 提供灵活的关系类型扩展机制
- 支持多时间段的关系存续期管理
- 为不同关系类型提供差异化的属性配置
- 支持以任意人物为中心构建和查看关系网
- 提供直观的关系可视化和查询功能
- **保障数据隐私**：优先本地存储，保护用户敏感的人际关系数据
- **灵活的数据共享**：支持公开关系网（如虚构人物）的分享机制

### 1.3 目标用户
- 需要管理复杂家族关系的用户
- 希望维护职场人脉的职场人士
- 需要记录同学、朋友信息的社交活跃人士
- 注重邻里关系维护的社区居民
- 文学爱好者（梳理小说、影视作品的人物关系）
- 历史研究者（整理历史人物关系）

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 关系类型管理
**功能描述**：系统采用插件化设计，支持动态扩展关系类型。

**功能点**：
- 预置关系类型（作为扩展示例）：
  - 亲属关系
  - 好友关系
  - 同事关系
  - 同学关系
  - 邻里关系
- 支持自定义新增关系类型
- 每种关系类型可配置：
  - 关系名称和图标
  - 专属属性字段（字段名、字段类型、是否必填）
  - 关系重要程度级别（如：核心、重要、一般）
  - 显示优先级

**优先级**：P0（最高）

#### 2.1.2 人物管理
**功能描述**：记录和管理关系网中的每个人物。人物可以是真实的人，也可以是虚构的角色（如小说、电影中的人物）。支持国际化，适配不同文化背景的人物信息管理。

**基础属性**（所有人物通用）：
- **姓名（必填）**：
  - 支持多种姓名格式：
    - 中文：姓 + 名（如：张三）
    - 西方：FirstName + MiddleName + LastName（如：John Michael Smith）
    - 日语：姓 + 名（如：山田太郎）
  - 系统根据文化背景提供对应的输入字段
  - 自动生成显示名称（displayName）
- **性别**：
  - 支持多元性别选项：
    - 男性（Male）
    - 女性（Female）
    - 其他（Other）
    - 非二元性别（Non-binary，部分地区）
    - 不愿透露（Prefer not to say）
  - 根据地区显示相应选项
- 生日（支持多种日期格式）
- **联系方式**：
  - 电话（支持国际格式，如：+86-138-0013-8000）
  - 微信号
  - 邮箱
  - 地址（支持国际地址格式）
- 头像
- 备注
- **文化背景**（culturalContext）：
  - 标识人物所属文化区域（zh-CN、en-US、ja-JP等）
  - 影响姓名格式、属性字段等
- 创建时间/修改时间

**虚构人物专属属性**：
- 是否虚构人物标识
- 来源（如：《红楼梦》、《权力的游戏》）
- 来源类型（小说/电影/电视剧/游戏/其他）

**功能点**：
- 添加/编辑/删除人物
- 人物搜索（支持姓名、电话、备注等字段）
- 人物分组查看（按关系类型、真实/虚构）
- 人物详情查看
- 人物导入/导出（支持微信好友导入）
- 标记可作为关系网中心的人物

**优先级**：P0

#### 2.1.3 关系建立与管理
**功能描述**：在人物之间建立具体的关系，支持多时间段。

**功能点**：
- 创建关系：选择两个人物，指定关系类型
- 关系属性配置：根据关系类型填写专属属性
- 多时间段管理：
  - 每段关系可记录起始时间和结束时间
  - 支持添加多个时间段（如：2010-2015同事，2018至今再次成为同事）
  - 标记当前活跃关系
- 关系强度标记（如：密切、一般、疏远）
- 关系备注（记录关系中的重要事件）
- 关系解除（软删除，保留历史）

**优先级**：P0

#### 2.1.4 关系可视化
**功能描述**：以图形化方式展示人际关系网络，支持以任意人物为中心。

**功能点**：
- 关系图谱展示（以选定人物为中心的网状图）
- **核心功能**：选择中心人物（可以是任意人物，不限于当前用户）
- 在图谱中点击任意人物节点，可切换为新的中心
- 按关系类型筛选显示
- 按关系时间段筛选
- 关系路径查询（查看两个人物之间的关系链）
- 关系统计面板：
  - 各类型关系数量
  - 活跃关系数量
  - 关系密度分析

**使用场景示例**：
- 场景1：以"我"为中心查看我的人际关系网
- 场景2：以"贾宝玉"为中心查看《红楼梦》中与他相关的人物关系
- 场景3：以"张三"为中心查看张三的社交网络
- 场景4：以"刘备"为中心查看三国时期的人物关系

**优先级**：P0

### 2.2 预置关系类型详细需求

#### 2.2.1 亲属关系
**专属属性**：
- 称谓（如：父亲、母亲、伯父、表姐等）
- 血缘关系（直系/旁系）
- 辈分
- 家庭地址
- 重要纪念日（结婚纪念日、忌日等）

**特点**：
- 通常为长期稳定关系
- 重视辈分和称谓
- 关注家庭活动和纪念日

#### 2.2.2 好友关系
**专属属性**：
- 认识场合（如：学校、工作、兴趣社群等）
- 共同爱好
- 友谊等级（普通朋友/好友/闺蜜/发小等）
- 最后联系时间
- 联系频率（经常/偶尔/很少）

**特点**：
- 关系强度变化较大
- 可能出现断联和重联
- 注重互动频率

#### 2.2.3 同事关系
**专属属性**：
- 公司名称
- 部门
- 职位
- 合作项目
- 工作关系（上级/下级/平级/跨部门）
- 办公地点

**特点**：
- 与工作变动强相关
- 多时间段特征明显（换工作导致关系中断/续接）
- 重视职业信息

#### 2.2.4 同学关系
**专属属性**：
- 学校名称
- 专业/班级
- 入学年份
- 毕业年份
- 同学类型（小学/初中/高中/大学/研究生）
- 是否同班/同寝

**特点**：
- 有明确的时间区间（就读期间）
- 毕业后可能疏远但仍保持同学身份
- 重视学校和班级信息

#### 2.2.5 邻里关系
**专属属性**：
- 小区名称
- 门牌号/楼栋单元
- 居住时间段
- 邻居类型（同楼层/同单元/同小区）
- 联系紧密度

**特点**：
- 与居住地变化相关
- 可能因搬家而中断
- 地理位置信息重要

### 2.3 辅助功能

#### 2.3.1 提醒功能
- 生日提醒
- 纪念日提醒
- 长期未联系提醒
- 自定义提醒事项

**优先级**：P1

#### 2.3.2 时间线功能
- 按时间轴查看关系变化历史
- 记录关系中的重要事件
- 支持事件备注和照片

**优先级**：P2

#### 2.3.3 数据存储与管理
**功能描述**：基于隐私保护的数据存储和管理策略。

**核心原则**：
- **隐私优先**：人际关系数据属于高度敏感的个人信息，默认仅存储在本地
- **分类存储**：
  - 私密关系网（个人真实人际关系）：默认本地存储
  - 公开关系网（虚构人物关系）：可选择云端同步和公开分享
- **用户可控**：用户完全控制数据存储位置和同步策略

**功能点**：

1. **本地存储（默认）**
   - 所有数据优先存储在小程序本地存储空间
   - 支持本地数据库（IndexedDB）存储结构化数据
   - 本地缓存图谱数据，提升性能

2. **云端同步（可选）**
   - 用户可选择将数据同步到云端备份
   - 明确提示哪些数据将被同步
   - 支持选择性同步（如仅同步公开关系网）

3. **数据导出**
   - **加密导出**：
     - 用户输入密码保护 → 生成加密压缩包（AES-256）
     - 适用于私密人际关系数据
     - 密码不存储在数据包中，仅用于加密
   - **明文导出**：
     - 用户不设置密码 → 生成普通压缩包
     - 适用于公开关系网（如《红楼梦》人物关系）
     - 方便分享和传播
   - 导出格式：JSON（压缩后，ZIP内部）
   - **文件扩展名**：`.gx[version]`
     - **gx**：关系（GuanXi）的缩写，g=关(guan)，x=系(xi)
     - **version**：数据格式版本号（0, 1, 2...）
     - 初始版本：`.gx0`
     - 当数据结构发生不兼容变更时，版本号递增
   - **文件命名规范**：`guanxi_[英文名]_[本地语言名]_[UTC时间].gx[version]`
     - `[英文名]`：可选，仅当本地语言非英语时提供，便于国际搜索
     - `[本地语言名]`：必填，关系网本地语言名称
     - `[UTC时间]`：UTC时区的导出日期时间（YYYYMMDDTHHmmssZ格式）
     - 示例：
       - `guanxi_我的关系网_20260321T083045Z.gx0`（仅中文）
       - `guanxi_hongloumeng_红楼梦_20260321T083045Z.gx0`（中英双语）
       - `guanxi_MyNetwork_20260321T083045Z.gx0`（英语环境）

4. **数据导入**
   - **版本识别**：通过文件扩展名（.gxN）快速识别数据格式版本
   - **版本兼容性检查**：
     - 支持的版本：直接导入或自动迁移到当前版本
     - 不支持的版本：提示用户升级程序
   - **智能识别**：自动检测导入文件是否加密
   - **加密文件导入流程**：
     - 检测到加密 → 提示用户输入密码
     - 密码错误 → 提示重试或取消
     - 密码正确 → 解密并导入数据
   - **明文文件导入流程**：
     - 直接解压并导入数据
   - **数据迁移**：旧版本数据自动迁移到新版本数据结构
   - **数据校验**：导入前验证数据格式和完整性
   - **冲突处理**：提供合并、覆盖、跳过等选项

5. **数据备份**
   - 支持定期自动备份到本地
   - 可手动触发备份
   - 备份文件加密存储

6. **数据清除**
   - 提供完全删除功能（清除本地和云端）
   - 删除前二次确认
   - 支持导出后删除

**技术实现要点**：
- 加密方案：使用 JSZip + AES 加密，或使用系统原生加密压缩
- 密码强度：要求至少8位，包含字母和数字
- 文件扩展名：`.gx[version]` 用于版本识别和兼容性管理
- 版本演进：不兼容更新时递增版本号（.gx0 → .gx1 → .gx2）
- 数据迁移：支持旧版本数据自动迁移到新版本
- 压缩率：预期压缩后大小为原数据的 20-30%

**优先级**：P0（数据隐私是核心需求）

#### 2.3.4 隐私与安全
- 数据加密存储
- 关系信息隐私保护
- 访问密码/指纹锁
- 隐私模式（隐藏敏感信息）

**优先级**：P1

## 3. 用户场景

### 3.1 场景一：新用户初次使用
1. 用户打开小程序，进入引导页
2. 选择关注的关系类型（亲戚、好友、同事等）
3. 可选择从微信好友导入基础人物
4. 开始添加人物并建立关系

### 3.2 场景二：记录复杂亲属关系
1. 用户进入"亲属关系"模块
2. 添加多个亲戚人物（父母、叔伯、堂兄弟等）
3. 为每个亲戚设置称谓、辈分等专属信息
4. 查看家族关系图谱，理清亲戚称谓
5. 可以任意一位长辈为中心，查看其人际关系网

### 3.3 场景三：管理同事关系变化
1. 用户在A公司工作期间添加同事
2. 离职后，关系时间段自动结束
3. 后来跳槽到B公司，与原同事再次成为同事
4. 添加新的关系时间段，保留完整工作关系史

### 3.4 场景四：维护好友关系
1. 定期查看"长期未联系"提醒
2. 查看好友详情，回顾过往互动
3. 更新最后联系时间和备注
4. 根据互动频率调整关系强度

### 3.5 场景五：梳理小说人物关系
1. 用户创建《红楼梦》人物关系网
2. 添加虚构人物：贾宝玉、林黛玉、薛宝钗等
3. 标记来源为《红楼梦》
4. 建立人物之间的关系（亲戚、朋友等）
5. 以贾宝玉为中心查看关系图谱
6. 切换到林黛玉为中心，查看不同视角的关系网
7. 查找贾宝玉与薛蟠之间的关系路径

### 3.6 场景六：研究历史人物关系
1. 历史研究者创建三国人物关系网
2. 添加历史人物：刘备、关羽、张飞、诸葛亮等
3. 建立君臣、结义兄弟等关系
4. 设置关系时间段（如：公元XXX年-XXX年）
5. 以刘备为中心查看蜀汉集团关系网
6. 切换到曹操为中心，对比魏国阵营

### 3.7 场景七：数据导出与分享
1. **私密数据导出**：
   - 用户整理完个人人际关系网
   - 选择"导出数据"
   - 输入密码（如："MySecret2024"）
   - 系统生成加密压缩包：`guanxi_我的关系网_20260321T083045Z.gx0`
   - 用户将文件保存到其他设备备份

2. **公开数据分享**：
   - 用户整理完《红楼梦》人物关系网
   - 选择"导出数据"
   - 可选择提供英文名："hongloumeng"
   - 不设置密码，直接导出
   - 系统生成普通压缩包：`guanxi_hongloumeng_红楼梦_20260321T083045Z.gx0`
   - 用户将文件分享到论坛或朋友圈，国际用户可通过"hongloumeng"搜索下载

### 3.8 场景八：数据导入与迁移
1. **导入加密数据**：
   - 新设备安装小程序
   - 选择"导入数据"
   - 选择之前导出的加密文件
   - 系统检测到文件已加密
   - 提示输入密码
   - 密码正确后成功导入所有数据

2. **导入公开数据**：
   - 用户从朋友处获得《红楼梦》关系网文件
   - 选择"导入数据"
   - 选择文件
   - 系统检测到文件未加密
   - 直接解压并导入
   - 提示导入成功，包含XX个人物、XX个关系

3. **数据合并**：
   - 本地已有部分数据
   - 导入新数据时检测到冲突
   - 系统提供选项：
     - 保留本地数据
     - 覆盖为导入数据
     - 合并（保留两者）
   - 用户选择合并，系统智能去重

### 3.9 场景九：国际用户使用体验
1. **英语用户添加人物**：
   - 用户打开小程序，系统检测到英语环境
   - 界面自动切换为英文
   - 添加人物时，姓名字段显示为：First Name、Middle Name、Last Name
   - 性别选项包括：Male、Female、Non-binary、Other、Prefer not to say
   - 日期格式为：MM/DD/YYYY

2. **跨文化关系网**：
   - 用户管理国际团队的人际关系
   - 添加中国同事：姓名格式选择"中文"，输入姓和名
   - 添加美国同事：姓名格式选择"西方"，输入 First/Middle/Last Name
   - 添加日本同事：姓名格式选择"日语"
   - 系统自动根据各人的文化背景正确显示姓名

3. **公开关系网的国际分享**：
   - 用户整理《三国演义》人物关系网
   - 导出时输入英文名："threekindoms"
   - 生成文件：`guanxi_threekindoms_三国演义_20260321T083045Z.gx0`
   - 上传到国际论坛
   - 英语用户搜索"threekindoms"可找到并下载
   - 导入后，界面根据用户语言显示（如显示为英文界面）

## 4. 界面原型要求

### 4.1 首页
- 顶部：搜索框、设置按钮
- 中部：关系类型快捷入口（卡片式）
- 下部：最近动态、生日提醒等快捷信息
- 快速操作：查看关系图谱（可选择中心人物）

### 4.2 人物列表页
- 顶部：搜索、筛选、添加按钮、语言切换
- 筛选选项：真实人物/虚构人物、按来源筛选、按文化背景筛选
- 列表：头像、显示名称、关系类型标签、关系数量
- 支持字母/拼音索引快速定位（根据语言自动切换）
- 操作菜单：编辑、删除、设为中心查看关系网

### 4.3 人物详情页
- 基础信息展示（根据文化背景显示对应字段）
- 姓名显示（支持多种格式）
- 性别显示（使用当前语言）
- 虚构人物显示来源信息
- 关系列表（可多个关系）
- 互动时间线
- 操作按钮（编辑、删除、分享）
- **核心按钮**：以此人物为中心查看关系网

### 4.4 关系图谱页
- 顶部：显示当前中心人物（本地化显示名称）、提供切换按钮、语言切换
- 中心节点：当前中心人物（突出显示）
- 周围节点：相关人物（按关系类型分区）
- 支持缩放、拖动
- 点击任意节点：查看详情或切换为新中心
- 筛选工具：关系类型、时间点
- 提示文字：根据语言本地化（如："点击人物可切换中心" / "Click to switch center"）
- **数据状态标识**：显示"本地数据"或"云端同步"

### 4.5 中心人物选择页
- 搜索框
- 常用中心人物列表（根据查看频率）
- 所有人物列表
- 显示每个人物的关系数量
- 点击选择后进入图谱页面

### 4.6 数据管理页面
- **语言设置**：
  - 界面语言选择（简体中文/繁体中文/English/日本語）
  - 日期格式偏好
  - 姓名显示格式偏好
- **存储策略设置**：
  - 本地存储（默认开启，不可关闭）
  - 云端同步（可选，默认关闭）
  - 显示本地数据大小
- **数据导出**：
  - 选择导出范围（全部/指定关系网）
  - 输入关系网本地语言名称（必填）
  - 输入关系网英文名称（可选，便于国际分享）
  - 密码保护开关
  - 输入密码（可选，至少8位）
  - 密码强度提示
  - 开始导出按钮
  - 导出进度显示
  - 导出完成后选择保存位置
- **数据导入**：
  - 选择文件
  - 自动识别加密状态和语言信息
  - 输入密码（如需要）
  - 数据预览（使用原始语言或当前语言显示）
  - 冲突处理选项（合并/覆盖/跳过）
  - 确认导入
  - 导入进度显示
- **数据备份**：
  - 自动备份设置（开/关、频率）
  - 手动立即备份
  - 备份历史记录
  - 恢复备份功能

## 6. 术语表

| 术语 | 定义 |
|------|------|
| 人物 | 关系网络中的个体，可以是真实的人，也可以是虚构的角色 |
| 关系类型 | 定义人与人之间关系的分类（如亲戚、好友等） |
| 关系实例 | 两个人物之间建立的具体关系 |
| Guanxi | 中文"关系"的拼音，比英文relation含义更丰富，包含情感、利益、社会地位等多重含义 |
| 时间段 | 关系存续的起止时间区间 |
| 关系强度 | 描述关系密切程度的指标 |
| 关系图谱 | 可视化展示人际关系网络的图形 |
| 中心人物 | 关系图谱的中心节点，可以是任意人物，不限于当前用户 |
| 专属属性 | 特定关系类型特有的属性字段 |
| 虚构人物 | 来自小说、电影、游戏等虚构作品中的角色 |
| 真实人物 | 现实生活中的真实人员 |
| 文化背景 | 人物所属的文化区域，影响姓名格式、属性字段等（如zh-CN、en-US） |
| nameFields | 国际化姓名字段，JSON对象，支持多种姓名格式 |
| displayName | 显示名称，根据姓名格式和用户偏好生成的用于界面显示的名称 |
| i18n | Internationalization（国际化）的缩写 |
| l10n | Localization（本地化）的缩写 |
