## 核心设计理念

### 1. 数据库驱动的类型系统

- **开放定义**：任何人都可以定义新的关系类型
- **数据库为唯一真实来源**：类型定义存储在 `guanxi_types` 集合
- **动态加载**：前端和后端运行时从数据库读取类型定义
- **即插即用**：添加新类型后立即可用，无需修改代码

### 2. 技术平等性

所有类型在技术上完全平等：
- 系统预装类型与用户自定义类型使用相同的数据结构
- 都存储在 `guanxi_types` 集合中
- 都可以被修改、删除、导入、导出
- 都遵循相同的集成方式和生命周期

### 3. 类型来源

- **系统预装**：首次安装时自动导入的5种基础类型
- **用户自定义**：用户根据自己需求创建的类型
- **社区共享**：从其他用户处导入的类型
- **AI生成**：通过AI辅助生成的类型

## 快速参考

### 关系类型优先级

优先级决定了在图谱中的显示重要程度（0-100）：

- **90+**：核心家庭（父母、配偶、子女）
- **80-89**：家庭关系（亲戚关系：85）
- **70-79**：亲密社交（好友关系：70）
- **60-69**：工作教育（同学：68，同事：65）
- **40-59**：一般社交（邻居：40）
- **30以下**：弱关系

### 常用字段类型

- `string` - 短文本（单行输入）
- `text` - 长文本（多行输入）
- `select` - 单选下拉
- `multiSelect` - 多选列表
- `number` - 数字
- `date` - 日期
- `boolean` - 开关
- `rating` - 评分（1-5星）

### 配置选项速查

| 配置项 | 说明 | 典型值 |
|--------|------|--------|
| `supportMultiPeriod` | 是否支持多时间段 | 好友、同事：true |
| `requirePeriod` | 是否要求填写时间段 | 同事、同学：true |
| `bidirectional` | 是否双向关系 | 大多数：true |
| `priority` | 显示优先级 | 40-85 |
| `allowDuplicate` | 是否允许重复关系 | 通常：false |
| `autoReminder` | 是否自动创建提醒 | 好友：true |
