**类型ID**：`work_colleague`
**分类**：`work`（工作关系）
**版本**：1.1
**设计日期**：2026-03-22

- 与用户自定义类型技术上完全平等
- 存储在 guanxi_types 集合中，可被修改或删除

## 1. 类型概述

### 1.1 业务场景

同事关系是职场中最重要的关系类型，涵盖了公司内外的工作合作关系。同事关系有明显的时间特征（入职-离职），且可能在换工作后中断。

**典型使用场景**：
- 记录当前和历史同事
- 管理跨公司的职场人脉
- 维护工作关系网络
- 职业发展参考（谁在哪个公司、什么职位）

### 1.2 设计目标

1. **明确组织信息**：公司、部门等组织结构信息
2. **记录工作关系**：上下级、平级、跨部门等关系类型
3. **时间段管理**：明确的入职-离职时间段
4. **合作项目追踪**：记录共同参与的项目

### 1.3 关键特点

- ✓ 支持多时间段（换工作会导致同事关系中断）
- ✓ 建议填写时间段（工作关系通常有明确起止时间）
- ✓ 双向关系（互为同事）
- ✓ 中高优先级（优先级：65）
- ✓ 7个专属字段：公司名称、部门、职位、合作关系、合作项目、工作地点、联系便利度

## 3. 字段详细说明

### 3.1 公司名称（company）

**字段名**：`company`
**类型**：`string`
**必填**：是

**说明**：
记录共同工作的公司名称，是同事关系最重要的标识信息。

**示例值**：
- "阿里巴巴集团"
- "腾讯科技（深圳）有限公司"
- "Microsoft China"
- "创业公司 - XX科技"

**验证规则**：
- 最大长度：100字符
- 不能为空

**使用建议**：
- 使用公司全称或常用简称
- 对于大公司可以包含分公司信息
- 创业公司可以标注"创业公司"

---

### 3.2 部门（department）

**字段名**：`department`
**类型**：`string`
**必填**：否

**说明**：
记录对方所在的部门，帮助定位组织结构中的位置。

**示例值**：
- "技术部"
- "研发中心-前端组"
- "市场部-华南区"
- "人力资源部"

**验证规则**：
- 最大长度：50字符

**使用建议**：
- 可以包含层级信息（如：研发中心-前端组）
- 跨部门合作时特别重要
- 如果不知道可以留空

---

### 3.3 职位（position）

**字段名**：`position`
**类型**：`string`
**必填**：否

**说明**：
记录对方的职位，有助于了解其职责范围和影响力。

**示例值**：
- "高级工程师"
- "产品经理"
- "技术总监"
- "CEO"
- "实习生"

**验证规则**：
- 最大长度：50字符

**注意**：
- 职位会随时间变化，应该记录在人物的"可变属性"（character_temporal_attributes）中
- 此处的职位是记录"当时共事时对方的职位"
- 如果对方升职了，人物的当前职位会更新，但关系中记录的仍是共事时的职位

---

### 3.4 合作关系（workRelation）

**字段名**：`workRelation`
**类型**：`select`
**必填**：是

**说明**：
明确双方在工作中的关系类型，是同事关系的核心区分字段。

**选项定义**：

| 值 | 中文 | 英文 | 说明 | 特点 |
|----|------|------|------|------|
| supervisor | 上级 | Supervisor | 对方是我的上级 | 汇报关系、非对等 |
| subordinate | 下级 | Subordinate | 对方是我的下级 | 管理关系、非对等 |
| peer_same_dept | 同部门同级 | Same Department Peer | 同部门的平级同事 | 最常见，密切合作 |
| peer_diff_dept | 跨部门同级 | Cross Department Peer | 不同部门的平级同事 | 跨部门协作 |
| partner | 合作伙伴 | Partner | 外部合作公司的人员 | 跨公司合作 |
| client | 客户/甲方 | Client | 服务的客户方 | 商务关系 |
| vendor | 供应商/乙方 | Vendor | 提供服务的供应商 | 商务关系 |

**默认值**：`peer_same_dept`（最常见）

**关系方向性**：
- `supervisor` 和 `subordinate` 是相对的
- 当A创建"上级"关系到B时，系统自动为B创建"下级"关系到A

---

### 3.5 合作项目（projects）

**字段名**：`projects`
**类型**：`text`（长文本）
**必填**：否

**说明**：
记录共同参与过的项目，有助于回忆合作经历和建立话题。

**示例值**：
- "XX电商平台改版, 移动端App开发"
- "2023年度营销活动策划"
- "公司年会组织"

**格式建议**：
- 多个项目用逗号或换行分隔
- 可以包含项目时间（如：2023年XX项目）
- 重要项目可以加粗或标星

**验证规则**：
- 最大长度：200字符

---

### 3.6 工作地点（workLocation）

**字段名**：`workLocation`
**类型**：`select`
**必填**：否

**说明**：
记录办公地点的相对位置，影响见面频率和沟通方式。

**选项定义**：

| 值 | 中文 | 英文 | 说明 |
|----|------|------|------|
| same_office | 同一办公室 | Same Office | 面对面工作，交流频繁 |
| same_building | 同一栋楼 | Same Building | 可以随时见面 |
| same_city | 同城不同办公地 | Same City | 需要特意约见 |
| remote | 异地/远程 | Remote | 主要通过线上沟通 |

**使用场景**：
- 影响关系强度计算（同办公室的同事关系通常更强）
- 影响沟通方式（远程同事主要线上沟通）
- 影响见面频率

---

### 3.7 联系便利度（contactEase）

**字段名**：`contactEase`
**类型**：`select`
**必填**：否

**说明**：
评估联系该同事的便利程度，考虑工作性质、时区、沟通习惯等因素。

**选项定义**：

| 值 | 中文 | 英文 | 说明 |
|----|------|------|------|
| very_easy | 非常方便 | Very Easy | 随时可以联系，响应及时 |
| easy | 比较方便 | Easy | 容易联系上，一般能及时回复 |
| moderate | 一般 | Moderate | 可以联系但需要预约或等待 |
| difficult | 不太方便 | Difficult | 不容易联系上，响应慢 |

**影响因素**：
- 工作地点：同办公室 > 同城 > 异地
- 时区差异：相同时区 > 跨时区
- 沟通习惯：习惯线上沟通 vs 只回邮件
- 工作性质：常在线 vs 外勤为主

**默认值**：`easy`

**帮助文本**：
- 中文："考虑工作性质、时区差异等因素"
- 英文："Consider work nature, time zone differences, etc."

## 4. 配置说明

### 4.1 时间段配置

```javascript
supportMultiPeriod: true,
requirePeriod: true
```

**理由**：
- 同事关系有明确的时间范围（入职到离职）
- 可能在多个公司与同一人共事（时间段会中断和恢复）
- 建议用户填写时间段，便于追溯职业历程

**典型场景**：
```javascript
// 场景1：曾在A公司共事，后来又在B公司重逢
periods: [
  { startTime: "2015-06-01", endTime: "2017-08-31" },  // A公司
  { startTime: "2020-03-01", endTime: null }           // B公司至今
]

// 场景2：当前在职
periods: [
  { startTime: "2023-01-15", endTime: null }
]

// 场景3：已离职
periods: [
  { startTime: "2018-06-01", endTime: "2021-05-31" }
]
```

---

### 4.2 关系方向配置

```javascript
bidirectional: true
```

**注意**：
- 虽然是双向关系，但 `workRelation` 字段是有方向性的
- A的"上级"对应B的"下级"
- 系统在创建双向关系时会自动调整workRelation的值

**实现逻辑**：
```javascript
// A创建关系：A → B (上级)
// 系统自动创建：B → A (下级)

const reverseWorkRelation = {
  'supervisor': 'subordinate',
  'subordinate': 'supervisor',
  'peer_same_dept': 'peer_same_dept',
  'peer_diff_dept': 'peer_diff_dept',
  'partner': 'partner',
  'client': 'vendor',
  'vendor': 'client'
};
```

---

### 4.3 优先级设置

```javascript
priority: 65
```

**理由**：
- 工作关系在职业发展中很重要
- 优先级65：高于普通社交关系（50）
- 低于亲密好友（70）和家庭关系（80+）
- 体现了工作关系在人际网络中的重要位置

---

### 4.4 自动提醒

```javascript
autoReminder: false
```

**理由**：
- 工作关系主要在工作时间接触，不需要额外提醒
- 如果离职后想保持联系，用户可以手动创建提醒
- 避免过多提醒骚扰用户

**例外情况**：
- 如果离职后想保持联系的重要同事，可以转为"好友关系"并启用提醒

## 6. 实现代码

### 6.1 类型类实现

```javascript
// types/work_colleague.type.js
const BaseGuanxiType = require('./base.type');

/**
 * 同事关系类型
 */
class WorkColleagueType extends BaseGuanxiType {
  constructor() {
    super({
      _id: 'work_colleague',
      name: '同事关系',
      nameEn: 'Colleague',
      nameI18nKey: 'guanxiType.work_colleague.name',
      icon: 'briefcase',
      color: '#95E1D3',
      category: 'work',
      description: '当前或曾经的同事、合作伙伴、业务联系人等工作关系',
      descriptionEn: 'Current or former colleagues, partners, and business contacts',

      fields: [
        // ... (完整字段定义见第2节)
      ],

      config: {
        supportMultiPeriod: true,
        requirePeriod: true,
        bidirectional: true,
        showInGraph: true,
        priority: 65,
        allowDuplicate: false,
        autoReminder: false
      }
    });
  }

  /**
   * 计算关系强度
   * 同事关系的强度由合作关系类型、工作地点、联系便利度共同决定
   */
  calculateStrength(attributes) {
    let strength = 40; // 基础分

    // 根据合作关系加分
    const relationScores = {
      supervisor: 35,        // 上级关系重要
      subordinate: 30,       // 下级关系重要
      peer_same_dept: 25,    // 同部门密切
      peer_diff_dept: 20,    // 跨部门一般
      partner: 25,           // 合作伙伴重要
      client: 30,            // 客户很重要
      vendor: 20             // 供应商一般
    };
    if (attributes.workRelation) {
      strength += relationScores[attributes.workRelation] || 0;
    }

    // 根据工作地点加分
    const locationScores = {
      same_office: 20,
      same_building: 15,
      same_city: 10,
      remote: 5
    };
    if (attributes.workLocation) {
      strength += locationScores[attributes.workLocation] || 0;
    }

    // 根据联系便利度加分
    const easeScores = {
      very_easy: 15,
      easy: 10,
      moderate: 5,
      difficult: 0
    };
    if (attributes.contactEase) {
      strength += easeScores[attributes.contactEase] || 0;
    }

    // 有合作项目额外加分
    if (attributes.projects && attributes.projects.trim().length > 0) {
      strength += 10;
    }

    return Math.min(strength, 100);
  }

  /**
   * 生成关系描述文本
   */
  generateDescription(attributes) {
    const parts = [];

    // 公司和部门
    if (attributes.company) {
      let orgInfo = attributes.company;
      if (attributes.department) {
        orgInfo += ` - ${attributes.department}`;
      }
      parts.push(orgInfo);
    }

    // 合作关系
    const relationLabels = {
      supervisor: '上级',
      subordinate: '下级',
      peer_same_dept: '同部门同事',
      peer_diff_dept: '跨部门同事',
      partner: '合作伙伴',
      client: '客户',
      vendor: '供应商'
    };
    if (attributes.workRelation) {
      parts.push(relationLabels[attributes.workRelation]);
    }

    // 职位
    if (attributes.position) {
      parts.push(`(${attributes.position})`);
    }

    return parts.join(' ');
  }

  /**
   * 获取反向关系类型
   * 用于创建双向关系时自动设置对方的合作关系
   */
  getReverseWorkRelation(workRelation) {
    const reverseMap = {
      'supervisor': 'subordinate',
      'subordinate': 'supervisor',
      'peer_same_dept': 'peer_same_dept',
      'peer_diff_dept': 'peer_diff_dept',
      'partner': 'partner',
      'client': 'vendor',
      'vendor': 'client'
    };
    return reverseMap[workRelation] || workRelation;
  }

  /**
   * 创建双向关系时的属性转换
   */
  transformAttributesForReverse(attributes) {
    const reversed = { ...attributes };

    // 反转合作关系
    if (attributes.workRelation) {
      reversed.workRelation = this.getReverseWorkRelation(attributes.workRelation);
    }

    // 其他字段保持不变
    return reversed;
  }

  /**
   * 验证属性值
   */
  validate(attributes) {
    const errors = [];

    // 必填验证
    if (!attributes.company || attributes.company.trim() === '') {
      errors.push({
        field: 'company',
        message: '公司名称不能为空'
      });
    }

    if (!attributes.workRelation) {
      errors.push({
        field: 'workRelation',
        message: '请选择合作关系'
      });
    }

    // 长度验证
    if (attributes.company && attributes.company.length > 100) {
      errors.push({
        field: 'company',
        message: '公司名称不能超过100个字符'
      });
    }

    if (attributes.department && attributes.department.length > 50) {
      errors.push({
        field: 'department',
        message: '部门名称不能超过50个字符'
      });
    }

    if (attributes.position && attributes.position.length > 50) {
      errors.push({
        field: 'position',
        message: '职位名称不能超过50个字符'
      });
    }

    if (attributes.projects && attributes.projects.length > 200) {
      errors.push({
        field: 'projects',
        message: '合作项目不能超过200个字符'
      });
    }

    return {
      valid: errors.length === 0,
      errors
    };
  }
}

module.exports = WorkColleagueType;
```

## 8. 验证规则

### 8.1 时间段验证

```javascript
function validatePeriods(periods, attributes) {
  const errors = [];

  // 同事关系建议填写时间段
  if (!periods || periods.length === 0) {
    errors.push({
      type: 'warning',
      message: '建议填写工作时间段，便于追溯职业历程'
    });
  }

  // 时间段不能重叠
  for (let i = 0; i < periods.length - 1; i++) {
    const current = periods[i];
    const next = periods[i + 1];

    if (current.endTime && next.startTime) {
      if (new Date(current.endTime) > new Date(next.startTime)) {
        errors.push({
          type: 'error',
          message: `时间段${i + 1}与时间段${i + 2}存在重叠`
        });
      }
    }
  }

  return errors;
}
```

### 8.2 业务逻辑验证

```javascript
function validateBusinessLogic(attributes) {
  const warnings = [];

  // 如果是上级/下级关系但在不同部门，给出提示
  if ((attributes.workRelation === 'supervisor' ||
       attributes.workRelation === 'subordinate') &&
      attributes.workLocation === 'remote') {
    warnings.push({
      type: 'info',
      message: '远程的上下级关系，建议记录沟通方式和时区信息'
    });
  }

  // 客户/供应商通常不在同一公司
  if ((attributes.workRelation === 'client' ||
       attributes.workRelation === 'vendor') &&
      attributes.workLocation === 'same_office') {
    warnings.push({
      type: 'warning',
      message: '客户/供应商通常不在同一办公室，请确认合作关系是否正确'
    });
  }

  return warnings;
}
```
