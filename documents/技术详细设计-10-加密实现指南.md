## 10. 加密实现指南

本章详细说明"关系网"小程序的数据加密机制，包括密码加密、数据导出加密、本地存储加密等安全措施的具体实现方法。

### 10.1 加密架构概述

#### 10.1.1 加密应用场景

```
┌─────────────────────────────────────────┐
│          数据加密应用场景                 │
├─────────────────────────────────────────┤
│ 1. 数据导出加密（密码保护的 .gx0 文件）  │
│ 2. 敏感字段加密（可选：联系方式等）       │
│ 3. 云同步传输加密（HTTPS + 额外加密层）  │
│ 4. 本地存储敏感数据加密（可选功能）       │
└─────────────────────────────────────────┘
```

#### 10.1.2 加密策略分层

| 层级 | 应用场景 | 加密方式 | 密钥管理 |
|-----|---------|---------|---------|
| **传输层** | 网络通信 | HTTPS (TLS) | 系统管理 |
| **应用层** | 数据导出 | AES-256-GCM | 用户密码派生 |
| **存储层** | 本地数据库 | 可选 AES-256 | 设备密钥 + 用户PIN |
| **字段层** | 敏感字段 | AES-256-CBC | 用户主密钥 |

### 10.2 核心加密库选择

#### 10.2.1 推荐加密库

**微信小程序环境**：
```javascript
// 使用微信小程序 crypto API（内置）
// 或引入第三方库：crypto-js

```

**Node.js 环境（云函数）**：
```javascript

```

#### 10.2.2 加密算法选择

| 算法 | 用途 | 密钥长度 | 特点 |
|-----|------|---------|------|
| **AES-256-GCM** | 数据导出加密 | 256位 | 认证加密，防篡改 |
| **AES-256-CBC** | 字段加密 | 256位 | 传统块加密 |
| **PBKDF2** | 密码派生 | - | 防暴力破解 |
| **HMAC-SHA256** | 数据完整性 | 256位 | 消息认证码 |
| **Scrypt** | 密码派生（高级）| - | 内存困难函数 |

### 10.3 密码派生（Key Derivation）

#### 10.3.1 PBKDF2 实现

```javascript
// utils/crypto/keyDerivation.js
import CryptoJS from 'crypto-js';

/**
 * 从用户密码派生加密密钥
 * @param {String} password - 用户密码
 * @param {String} salt - 盐值（应随机生成并保存）
 * @param {Number} iterations - 迭代次数（建议 100000+）
 * @param {Number} keySize - 密钥大小（字节）
 * @returns {String} 派生的密钥（Hex格式）
 */
export function deriveKey(password, salt, iterations = 100000, keySize = 32) {
  try {
    // 使用 PBKDF2 派生密钥
    const key = CryptoJS.PBKDF2(password, salt, {
      keySize: keySize / 4,  // CryptoJS 使用 32位字 为单位
      iterations: iterations,
      hasher: CryptoJS.algo.SHA256
    });
    
    return key.toString(CryptoJS.enc.Hex);
  } catch (error) {
    console.error('密钥派生失败:', error);
    throw new Error('密钥派生失败');
  }
}

/**
 * 生成随机盐值
 * @param {Number} length - 盐值长度（字节）
 * @returns {String} 盐值（Hex格式）
 */
export function generateSalt(length = 16) {
  const randomWords = CryptoJS.lib.WordArray.random(length);
  return randomWords.toString(CryptoJS.enc.Hex);
}

/**
 * 验证密码是否正确
 * @param {String} password - 用户输入的密码
 * @param {String} salt - 保存的盐值
 * @param {String} expectedKey - 预期的密钥（用于验证）
 * @param {Number} iterations - 迭代次数
 * @returns {Boolean} 密码是否正确
 */
export function verifyPassword(password, salt, expectedKey, iterations = 100000) {
  const derivedKey = deriveKey(password, salt, iterations);
  return derivedKey === expectedKey;
}
```

#### 10.3.2 Scrypt 实现（高级）

```javascript
// utils/crypto/scrypt.js
// 需要引入 scrypt.js 库
import scrypt from 'scrypt-js';

/**
 * 使用 Scrypt 派生密钥（更安全，但更慢）
 * @param {String} password - 用户密码
 * @param {String} salt - 盐值
 * @param {Number} N - CPU/内存成本参数（建议 16384）
 * @param {Number} r - 块大小参数（建议 8）
 * @param {Number} p - 并行化参数（建议 1）
 * @param {Number} dkLen - 密钥长度（字节）
 * @returns {Promise<String>} 派生的密钥（Hex格式）
 */
export async function deriveKeyScrypt(password, salt, N = 16384, r = 8, p = 1, dkLen = 32) {
  try {
    const passwordBuffer = Buffer.from(password, 'utf8');
    const saltBuffer = Buffer.from(salt, 'hex');
    
    const key = await scrypt.scrypt(passwordBuffer, saltBuffer, N, r, p, dkLen);
    return Buffer.from(key).toString('hex');
  } catch (error) {
    console.error('Scrypt 密钥派生失败:', error);
    throw new Error('密钥派生失败');
  }
}
```

### 10.4 数据导出加密

#### 10.4.1 加密导出流程

```
用户输入密码
    ↓
生成随机盐值 (Salt)
    ↓
派生加密密钥 (Key) ← PBKDF2(密码, 盐值, 100000次)
    ↓
生成随机初始化向量 (IV)
    ↓
序列化数据 (JSON)
    ↓
AES-256-GCM 加密
    ↓
计算 HMAC-SHA256（可选，额外完整性检查）
    ↓
打包文件: { salt, iv, authTag, encryptedData, hmac }
    ↓
保存为 .gx0 文件
```

#### 10.4.2 加密实现

```javascript
// utils/crypto/encryption.js
import CryptoJS from 'crypto-js';
import { deriveKey, generateSalt } from './keyDerivation';

/**
 * 加密数据导出
 * @param {Object} data - 要导出的数据对象
 * @param {String} password - 用户设置的密码
 * @returns {Object} 加密后的数据包
 */
export async function encryptExportData(data, password) {
  try {
    // 1. 生成随机盐值
    const salt = generateSalt(16);
    
    // 2. 派生加密密钥
    const key = deriveKey(password, salt, 100000, 32);
    
    // 3. 序列化数据
    const jsonData = JSON.stringify(data);
    
    // 4. 生成随机 IV（初始化向量）
    const iv = CryptoJS.lib.WordArray.random(12);  // GCM 模式使用 12 字节 IV
    
    // 5. AES-256-GCM 加密
    const encrypted = CryptoJS.AES.encrypt(jsonData, CryptoJS.enc.Hex.parse(key), {
      iv: iv,
      mode: CryptoJS.mode.GCM,
      padding: CryptoJS.pad.NoPadding
    });
    
    // 6. 提取密文和认证标签
    const ciphertext = encrypted.ciphertext.toString(CryptoJS.enc.Base64);
    const authTag = encrypted.tag ? encrypted.tag.toString(CryptoJS.enc.Base64) : '';
    
    // 7. 计算 HMAC（额外的完整性保护）
    const hmac = CryptoJS.HmacSHA256(ciphertext, key).toString(CryptoJS.enc.Hex);
    
    // 8. 返回加密数据包
    return {
      version: '1.0',
      algorithm: 'AES-256-GCM',
      kdf: {
        algorithm: 'PBKDF2-SHA256',
        iterations: 100000,
        salt: salt
      },
      encryption: {
        iv: iv.toString(CryptoJS.enc.Base64),
        authTag: authTag,
        ciphertext: ciphertext
      },
      integrity: {
        hmac: hmac
      },
      metadata: {
        encryptedAt: new Date().toISOString(),
        dataSize: jsonData.length
      }
    };
  } catch (error) {
    console.error('数据加密失败:', error);
    throw new Error('数据加密失败');
  }
}

/**
 * 解密数据导入
 * @param {Object} encryptedPackage - 加密的数据包
 * @param {String} password - 用户输入的密码
 * @returns {Object} 解密后的原始数据
 */
export async function decryptImportData(encryptedPackage, password) {
  try {
    const { version, kdf, encryption, integrity } = encryptedPackage;
    
    // 1. 验证版本
    if (version !== '1.0') {
      throw new Error('不支持的加密版本');
    }
    
    // 2. 派生解密密钥
    const key = deriveKey(password, kdf.salt, kdf.iterations, 32);
    
    // 3. 验证 HMAC（可选，但推荐）
    if (integrity && integrity.hmac) {
      const calculatedHmac = CryptoJS.HmacSHA256(
        encryption.ciphertext,
        key
      ).toString(CryptoJS.enc.Hex);
      
      if (calculatedHmac !== integrity.hmac) {
        throw new Error('数据完整性验证失败，文件可能已被篡改');
      }
    }
    
    // 4. AES-256-GCM 解密
    const decrypted = CryptoJS.AES.decrypt(
      {
        ciphertext: CryptoJS.enc.Base64.parse(encryption.ciphertext),
        tag: encryption.authTag ? CryptoJS.enc.Base64.parse(encryption.authTag) : undefined
      },
      CryptoJS.enc.Hex.parse(key),
      {
        iv: CryptoJS.enc.Base64.parse(encryption.iv),
        mode: CryptoJS.mode.GCM,
        padding: CryptoJS.pad.NoPadding
      }
    );
    
    // 5. 转换为字符串
    const decryptedString = decrypted.toString(CryptoJS.enc.Utf8);
    
    if (!decryptedString) {
      throw new Error('解密失败，密码可能不正确');
    }
    
    // 6. 解析 JSON
    const data = JSON.parse(decryptedString);
    
    return data;
  } catch (error) {
    console.error('数据解密失败:', error);
    
    // 区分不同的错误类型
    if (error.message.includes('解密失败') || error.message.includes('密码')) {
      throw new Error('密码错误或文件已损坏');
    } else if (error.message.includes('完整性验证失败')) {
      throw new Error('文件完整性验证失败，可能已被篡改');
    } else {
      throw new Error('数据解密失败');
    }
  }
}
```

#### 10.4.3 在导出服务中集成

```javascript
// services/exportService.js
import { encryptExportData, decryptImportData } from '../utils/crypto/encryption';
import { generateExportFileName } from '../utils/i18n/exportFormatter';

/**
 * 导出数据（支持加密）
 * @param {Object} options - 导出选项
 * @returns {String} 导出的文件路径
 */
export async function exportData(options) {
  const {
    localName,
    englishName,
    password,  // 可选：加密密码
    exportRange = 'all',
    selectedIds = []
  } = options;
  
  try {
    // 1. 收集要导出的数据
    let data = await collectExportData(exportRange, selectedIds);
    
    // 2. 添加元数据
    data = {
      meta: {
        version: '1.0.0',
        exportTime: new Date().toISOString(),
        locale: i18n.locale,
        name: {
          local: localName,
          english: englishName
        },
        encrypted: !!password
      },
      data: data
    };
    
    // 3. 如果提供密码，则加密
    let finalData;
    if (password) {
      finalData = await encryptExportData(data, password);
    } else {
      finalData = data;
    }
    
    // 4. 生成文件名
    const fileName = generateExportFileName(localName, englishName);
    
    // 5. 保存文件
    const fileData = JSON.stringify(finalData, null, 2);
    const filePath = `${wx.env.USER_DATA_PATH}/${fileName}`;
    
    const fs = wx.getFileSystemManager();
    fs.writeFileSync(filePath, fileData, 'utf8');
    
    return {
      success: true,
      filePath: filePath,
      fileName: fileName,
      encrypted: !!password
    };
  } catch (error) {
    console.error('导出失败:', error);
    throw error;
  }
}

/**
 * 导入数据（支持解密）
 * @param {String} filePath - 导入文件路径
 * @param {String} password - 解密密码（如果文件已加密）
 * @returns {Object} 导入结果
 */
export async function importData(filePath, password = null) {
  try {
    // 1. 读取文件
    const fs = wx.getFileSystemManager();
    const fileData = fs.readFileSync(filePath, 'utf8');
    const parsedData = JSON.parse(fileData);
    
    // 2. 检查是否加密
    let data;
    if (parsedData.algorithm && parsedData.encryption) {
      // 文件已加密
      if (!password) {
        throw new Error('文件已加密，请输入密码');
      }
      
      // 解密
      data = await decryptImportData(parsedData, password);
    } else if (parsedData.meta && parsedData.meta.encrypted) {
      throw new Error('文件已加密，但格式不正确');
    } else {
      // 明文文件
      data = parsedData;
    }
    
    // 3. 验证数据格式
    if (!data.meta || !data.data) {
      throw new Error('文件格式不正确');
    }
    
    // 4. 导入数据到本地数据库
    const result = await importDataToDatabase(data.data);
    
    return {
      success: true,
      imported: result,
      metadata: data.meta
    };
  } catch (error) {
    console.error('导入失败:', error);
    throw error;
  }
}
```

### 10.5 敏感字段加密

#### 10.5.1 字段级加密实现

```javascript
// utils/crypto/fieldEncryption.js

/**
 * 加密单个字段
 * @param {String} plaintext - 明文
 * @param {String} masterKey - 主密钥
 * @returns {String} 加密后的字段（Base64格式）
 */
export function encryptField(plaintext, masterKey) {
  if (!plaintext) return plaintext;
  
  try {
    // 生成随机 IV
    const iv = CryptoJS.lib.WordArray.random(16);
    
    // AES-256-CBC 加密
    const encrypted = CryptoJS.AES.encrypt(plaintext, masterKey, {
      iv: iv,
      mode: CryptoJS.mode.CBC,
      padding: CryptoJS.pad.Pkcs7
    });
    
    // 组合 IV 和密文
    const combined = iv.concat(encrypted.ciphertext);
    return combined.toString(CryptoJS.enc.Base64);
  } catch (error) {
    console.error('字段加密失败:', error);
    return plaintext;  // 加密失败时返回原文（可根据需求调整）
  }
}

/**
 * 解密单个字段
 * @param {String} ciphertext - 密文（Base64格式）
 * @param {String} masterKey - 主密钥
 * @returns {String} 解密后的明文
 */
export function decryptField(ciphertext, masterKey) {
  if (!ciphertext) return ciphertext;
  
  try {
    // 解析 Base64
    const combined = CryptoJS.enc.Base64.parse(ciphertext);
    
    // 提取 IV（前 16 字节）和密文
    const iv = CryptoJS.lib.WordArray.create(combined.words.slice(0, 4));
    const encrypted = CryptoJS.lib.WordArray.create(combined.words.slice(4));
    
    // AES-256-CBC 解密
    const decrypted = CryptoJS.AES.decrypt(
      { ciphertext: encrypted },
      masterKey,
      {
        iv: iv,
        mode: CryptoJS.mode.CBC,
        padding: CryptoJS.pad.Pkcs7
      }
    );
    
    return decrypted.toString(CryptoJS.enc.Utf8);
  } catch (error) {
    console.error('字段解密失败:', error);
    return '[解密失败]';
  }
}

/**
 * 加密对象中的指定字段
 * @param {Object} obj - 对象
 * @param {Array<String>} fields - 需要加密的字段列表
 * @param {String} masterKey - 主密钥
 * @returns {Object} 加密后的对象
 */
export function encryptObjectFields(obj, fields, masterKey) {
  const encrypted = { ...obj };
  
  fields.forEach(field => {
    if (obj[field]) {
      encrypted[field] = encryptField(obj[field], masterKey);
    }
  });
  
  return encrypted;
}

/**
 * 解密对象中的指定字段
 * @param {Object} obj - 对象
 * @param {Array<String>} fields - 需要解密的字段列表
 * @param {String} masterKey - 主密钥
 * @returns {Object} 解密后的对象
 */
export function decryptObjectFields(obj, fields, masterKey) {
  const decrypted = { ...obj };
  
  fields.forEach(field => {
    if (obj[field]) {
      decrypted[field] = decryptField(obj[field], masterKey);
    }
  });
  
  return decrypted;
}
```

### 10.6 密钥管理

#### 10.6.1 主密钥生成和存储

```javascript
// utils/crypto/keyManager.js

/**
 * 初始化用户主密钥
 * @param {String} userPin - 用户 PIN 码（4-6位数字）
 * @returns {String} 生成的主密钥
 */
export async function initializeMasterKey(userPin) {
  try {
    // 1. 检查是否已存在主密钥
    const existingKey = wx.getStorageSync('_mk');
    if (existingKey) {
      throw new Error('主密钥已存在');
    }
    
    // 2. 生成随机主密钥（256位）
    const masterKey = CryptoJS.lib.WordArray.random(32).toString(CryptoJS.enc.Hex);
    
    // 3. 使用设备密钥 + 用户 PIN 加密主密钥
    const deviceKey = await getDeviceKey();
    const pinKey = deriveKey(userPin, deviceKey, 10000, 32);
    
    const encryptedMasterKey = CryptoJS.AES.encrypt(masterKey, pinKey).toString();
    
    // 4. 保存加密后的主密钥
    wx.setStorageSync('_mk', encryptedMasterKey);
    wx.setStorageSync('_mk_salt', deviceKey);
    
    return masterKey;
  } catch (error) {
    console.error('初始化主密钥失败:', error);
    throw error;
  }
}

/**
 * 获取用户主密钥
 * @param {String} userPin - 用户 PIN 码（可选，缓存中有则不需要）
 * @returns {String|null} 主密钥，如果未启用加密则返回 null
 */
export async function getMasterKey(userPin = null) {
  try {
    // 1. 检查是否启用字段加密
    const encryptionEnabled = wx.getStorageSync('_enc_enabled');
    if (!encryptionEnabled) {
      return null;
    }
    
    // 2. 检查缓存
    const cachedKey = getApp().globalData.masterKey;
    if (cachedKey) {
      return cachedKey;
    }
    
    // 3. 如果没有缓存，需要 PIN 码
    if (!userPin) {
      throw new Error('需要输入 PIN 码');
    }
    
    // 4. 读取加密的主密钥
    const encryptedMasterKey = wx.getStorageSync('_mk');
    const salt = wx.getStorageSync('_mk_salt');
    
    if (!encryptedMasterKey || !salt) {
      throw new Error('主密钥不存在');
    }
    
    // 5. 派生 PIN 密钥
    const pinKey = deriveKey(userPin, salt, 10000, 32);
    
    // 6. 解密主密钥
    const decrypted = CryptoJS.AES.decrypt(encryptedMasterKey, pinKey);
    const masterKey = decrypted.toString(CryptoJS.enc.Utf8);
    
    if (!masterKey) {
      throw new Error('PIN 码错误');
    }
    
    // 7. 缓存主密钥（存储在内存中）
    getApp().globalData.masterKey = masterKey;
    
    return masterKey;
  } catch (error) {
    console.error('获取主密钥失败:', error);
    throw error;
  }
}

/**
 * 获取或生成设备密钥
 * @returns {String} 设备密钥
 */
async function getDeviceKey() {
  // 尝试从存储读取
  let deviceKey = wx.getStorageSync('_device_key');
  
  if (!deviceKey) {
    // 生成新的设备密钥
    deviceKey = CryptoJS.lib.WordArray.random(16).toString(CryptoJS.enc.Hex);
    wx.setStorageSync('_device_key', deviceKey);
  }
  
  return deviceKey;
}

/**
 * 清除主密钥缓存（用户锁定应用时调用）
 */
export function clearMasterKeyCache() {
  if (getApp().globalData) {
    delete getApp().globalData.masterKey;
  }
}

/**
 * 修改 PIN 码
 * @param {String} oldPin - 旧 PIN 码
 * @param {String} newPin - 新 PIN 码
 */
export async function changePIN(oldPin, newPin) {
  try {
    // 1. 验证旧 PIN 并获取主密钥
    const masterKey = await getMasterKey(oldPin);
    
    if (!masterKey) {
      throw new Error('旧 PIN 码错误');
    }
    
    // 2. 使用新 PIN 重新加密主密钥
    const deviceKey = await getDeviceKey();
    const newPinKey = deriveKey(newPin, deviceKey, 10000, 32);
    const encryptedMasterKey = CryptoJS.AES.encrypt(masterKey, newPinKey).toString();
    
    // 3. 保存
    wx.setStorageSync('_mk', encryptedMasterKey);
    
    // 4. 清除缓存
    clearMasterKeyCache();
    
    return true;
  } catch (error) {
    console.error('修改 PIN 失败:', error);
    throw error;
  }
}
```

```javascript
// pages/security/pin-input/pin-input.js
Page({
  data: {
    title: '输入 PIN 码',
    subtitle: '请输入您的 6 位 PIN 码',
    pin: '',
    pinLength: 6,
    keyboard: ['1', '2', '3', '4', '5', '6', '7', '8', '9', '0']
  },
  
  onKeyPress(e) {
    const key = e.currentTarget.dataset.key;
    const currentPin = this.data.pin;
    
    if (currentPin.length < this.data.pinLength) {
      const newPin = currentPin + key;
      this.setData({ pin: newPin });
      
      // 如果 PIN 输入完成，自动验证
      if (newPin.length === this.data.pinLength) {
        this.verifyPin(newPin);
      }
    }
  },
  
  onDelete() {
    const currentPin = this.data.pin;
    if (currentPin.length > 0) {
      this.setData({
        pin: currentPin.slice(0, -1)
      });
    }
  },
  
  async verifyPin(pin) {
    try {
      wx.showLoading({ title: '验证中...' });
      
      // 尝试使用 PIN 获取主密钥
      const masterKey = await getMasterKey(pin);
      
      wx.hideLoading();
      
      if (masterKey) {
        // 验证成功，返回上一页
        wx.navigateBack();
      } else {
        // 验证失败
        this.handlePinError();
      }
    } catch (error) {
      wx.hideLoading();
      this.handlePinError();
    }
  },
  
  handlePinError() {
    wx.showToast({
      title: 'PIN 码错误',
      icon: 'none'
    });
    
    this.setData({ pin: '' });
  }
});
```

### 10.7 性能优化

#### 10.7.1 加密性能优化策略

1. **延迟加密**：仅在用户需要时才加密敏感字段
2. **批量操作**：一次性加密/解密多个字段
3. **缓存主密钥**：避免重复派生密钥
4. **Web Worker**：在后台线程执行加密操作（小程序 Worker）

```javascript
// workers/crypto.worker.js
// 微信小程序 Worker 示例

worker.onMessage(function(message) {
  const { action, data, key } = message;
  
  if (action === 'encrypt') {
    // 批量加密
    const results = data.map(item => {
      return CryptoJS.AES.encrypt(item, key).toString();
    });
    
    worker.postMessage({
      action: 'encrypt',
      results: results
    });
  } else if (action === 'decrypt') {
    // 批量解密
    const results = data.map(item => {
      const decrypted = CryptoJS.AES.decrypt(item, key);
      return decrypted.toString(CryptoJS.enc.Utf8);
    });
    
    worker.postMessage({
      action: 'decrypt',
      results: results
    });
  }
});
```

### 10.8 安全最佳实践

#### 10.8.1 密钥管理最佳实践

- ✅ **不要硬编码密钥**：永远不要在代码中硬编码加密密钥
- ✅ **使用强密码派生**：PBKDF2 迭代次数至少 100,000 次
- ✅ **随机盐值**：每次加密使用新的随机盐值
- ✅ **随机 IV**：每次加密使用新的随机 IV
- ✅ **安全存储**：使用设备密钥 + 用户 PIN 保护主密钥
- ✅ **定期清除缓存**：应用进入后台时清除内存中的密钥

#### 10.8.2 加密实现最佳实践

- ✅ **使用认证加密**：优先使用 AES-GCM 而非 AES-CBC
- ✅ **完整性保护**：使用 HMAC 验证数据完整性
- ✅ **版本控制**：在加密数据中包含版本号，便于未来升级
- ✅ **错误处理**：不要泄露加密错误的详细信息
- ✅ **安全删除**：删除敏感数据时覆写内存

#### 10.8.3 用户体验最佳实践

- ✅ **密码强度提示**：引导用户设置强密码
- ✅ **加密状态提示**：明确告知用户哪些数据已加密
- ✅ **性能优化**：避免加密影响用户体验
- ✅ **密码恢复**：提供安全的密码恢复机制（如安全问题）
- ✅ **透明性**：向用户说明加密的作用和重要性
