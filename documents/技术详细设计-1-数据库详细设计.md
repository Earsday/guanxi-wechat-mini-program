## 1. 数据库详细设计

### 1.1 集合结构

#### 1.1.1 users（用户集合）
存储小程序用户信息。

```javascript
{
  _id: ObjectId,              // 系统生成的用户ID
  _openid: String,            // 微信用户唯一标识（云开发自动添加）
  nickName: String,           // 昵称
  avatarUrl: String,          // 头像URL
  phone: String,              // 手机号（可选）
  settings: {                 // 用户设置
    theme: String,            // 主题
    defaultView: String,      // 默认视图
    reminderEnabled: Boolean  // 是否启用提醒
  },
  createdAt: Date,            // 创建时间
  updatedAt: Date             // 更新时间
}
```

**索引**：
- `_openid`: 唯一索引
- `createdAt`: 普通索引

#### 1.1.2 characters（人物集合）
存储关系网中的所有人物信息。

```javascript
{
  _id: ObjectId,              // 人物ID
  userId: String,             // 所属用户ID（openid）
  name: String,               // 姓名（必填）
  alias: [String],            // 别名/昵称列表
  gender: String,             // 性别: male/female/other

  // 生日（支持阴阳历）
  birthday: {
    solar: String,            // 阳历生日 (YYYY-MM-DD)
    lunar: String,            // 阴历生日 (YYYY-MM-DD)，可选
    primary: String           // 主要使用：'solar'/'lunar'
  },

  // 联系方式
  contacts: {
    // 电话（最多3个）
    phones: [{
      number: String,         // 电话号码（支持国际格式）
      label: String,          // 标签：primary/backup1/backup2 或 home/work/mobile
      isPrimary: Boolean      // 是否主号码
    }],

    // 邮箱（最多2个）
    emails: [{
      address: String,        // 邮箱地址
      type: String            // 类型：work/personal
    }],

    // 地址（最多2个）
    addresses: [{
      address: String,        // 详细地址
      type: String,           // 类型：work/home
      postalCode: String,     // 邮政编码（可选）
      country: String,        // 国家
      province: String,       // 省/州
      city: String            // 城市
    }],

    // 即时通讯账号
    im: {
      wechatId: String,       // 微信号
      qq: String,             // QQ号
      douyin: String,         // 抖音号
      tiktok: String,         // TikTok号
      whatsapp: String,       // WhatsApp
      telegram: String,       // Telegram
      line: String,           // LINE（日本）
      kakao: String           // KakaoTalk（韩国）
    }
  },

  avatar: String,             // 头像URL（云存储路径）
  description: String,        // 人物描述
  tags: [String],             // 标签

  // 固定属性（出生时确定，不会改变）
  immutableAttributes: {
    birthPlace: String,       // 出生地
    birthNationality: String, // 出生时国籍
    ethnicity: String,        // 民族
    zodiacSign: String,       // 星座（根据阳历生日自动计算）
    chineseZodiac: String,    // 生肖（根据农历年份自动计算）
    bloodType: String         // 血型：A/B/AB/O/unknown
  },

  // 虚构人物标识
  isFictional: Boolean,       // 是否虚构人物（默认false）
  source: String,             // 来源（如：《红楼梦》、《权力的游戏》）

  // 系统字段
  isUser: Boolean,            // 是否是当前用户本人（每个用户只能有一个）
  importFrom: String,         // 导入来源：wechat/manual/import
  createdAt: Date,
  updatedAt: Date,

  // 统计字段（冗余，提升查询性能）
  stats: {
    guanxiCount: Number,      // 关系数量
    activeGuanxiCount: Number // 活跃关系数量
  }
}
```

**索引**：
- `userId`: 普通索引
- `{userId: 1, name: 1}`: 复合索引
- `{userId: 1, isUser: 1}`: 复合索引（查找用户代表人物）
- `name`: 文本索引（支持搜索）
- `contacts.phones.number`: 普通索引
- `contacts.emails.address`: 普通索引
- `tags`: 普通索引

**约束**：
- 每个用户只能有一个 `isUser: true` 的人物
- name 不能为空
- `contacts.phones` 数组最多3个元素
- `contacts.emails` 数组最多2个元素
- `contacts.addresses` 数组最多2个元素

#### 1.1.3 character_temporal_attributes（人物可变属性集合）
存储人物的可变属性及其时间段。某些属性在人物一生中可能会变化，需要记录变化的时间段。

```javascript
{
  _id: ObjectId,
  userId: String,             // 所属用户ID
  characterId: ObjectId,      // 人物ID

  // 属性类型
  attributeType: String,      // 属性类型：nationality/residence/occupation/politicalParty/religion

  // 属性值
  value: String,              // 属性值

  // 时间段
  startTime: String,          // 生效开始时间 (YYYY-MM-DD)
  endTime: String,            // 生效结束时间 (YYYY-MM-DD)，null表示持续中
  isActive: Boolean,          // 是否当前生效

  // 备注
  note: String,               // 变更备注（如：入籍原因、换工作原因）

  // 系统字段
  createdAt: Date,
  updatedAt: Date
}
```

**索引**：
- `{userId: 1, characterId: 1}`: 复合索引
- `{characterId: 1, attributeType: 1}`: 复合索引
- `{characterId: 1, isActive: 1}`: 复合索引

**约束**：
- 同一人物的同一属性类型，不能有时间重叠的记录
- startTime 不能为空
- 同一属性类型，最多只能有一个 isActive: true 的记录

**支持的属性类型**：
- `nationality`: 国籍（可能因入籍/归化而改变）
- `residence`: 居住地（可能因搬迁而改变）
- `occupation`: 职业（可能因换工作而改变）
- `politicalParty`: 党派（可能因政治立场变化而改变）
- `religion`: 宗教信仰（可能因信仰变化而改变）

#### 1.1.4 guanxi_types（关系类型集合）

**集合说明**：

关系类型集合是系统的**唯一类型定义来源**。所有关系类型（无论是开发者提供的示例类型，还是用户自定义的类型）都存储在此集合中。系统在运行时从该集合读取类型定义，动态生成表单和验证规则。

**设计理念**：
- 📊 **数据库驱动**：类型定义存储在数据库，前后端统一读取
- 🔓 **开放扩展**：任何人都可以定义新的关系类型
- 🤝 **社区协作**：类型定义可导出分享，促进社区共建
- 🎨 **动态加载**：无需修改代码，添加类型即可使用

**集合结构**：

```javascript
{
  _id: String,                // 类型ID（全局唯一，如：family_relative）
  name: String,               // 类型名称
  nameEn: String,             // 英文名称
  icon: String,               // 图标标识
  color: String,              // 主题色（HEX格式）
  category: String,           // 分类：social/family/work/education/neighbor/custom
  description: String,        // 描述
  descriptionEn: String,      // 英文描述

  // 字段定义
  fields: [{
    name: String,             // 字段名
    label: String,            // 字段标签
    labelEn: String,          // 英文标签
    type: String,             // 字段类型：string/number/date/select/multiSelect/boolean
    required: Boolean,        // 是否必填
    defaultValue: Any,        // 默认值
    options: [String],        // 选项（type为select时）
    placeholder: String,      // 占位符
    validation: {             // 验证规则
      min: Number,
      max: Number,
      pattern: String,
      message: String
    },
    displayOrder: Number      // 显示顺序
  }],

  // 配置
  config: {
    supportMultiPeriod: Boolean,    // 是否支持多时间段（默认true）
    requirePeriod: Boolean,         // 是否必须有时间段（默认false）
    bidirectional: Boolean,         // 是否双向关系（默认true）
    showInGraph: Boolean,           // 是否在图谱中显示（默认true）
    priority: Number,               // 显示优先级（0-100）

    // 关系推导配置（可选）
    enableDeduction: Boolean,       // 是否启用关系推导
    deductionRules: Array,          // 推导规则定义
    deductionDepth: Number,         // 单次推导深度
    deductionStrategy: String,      // 推导策略
    autoCreateDeduced: Boolean,     // 是否自动创建推导关系
    deductionConfidenceThreshold: Number,
    deduplicationEnabled: Boolean,
    maxChainLength: Number
  },

  // 国际化资源（可选）
  i18n: {
    'zh-CN': Object,          // 简体中文翻译
    'en-US': Object,          // 英文翻译
    'zh-TW': Object           // 繁体中文翻译（可选）
  },

  // 元数据
  version: String,            // 类型定义版本（如：1.0.0）
  author: String,             // 作者（用户ID或组织名）
  isEnabled: Boolean,         // 是否启用（默认true）
  isPublic: Boolean,          // 是否公开分享（默认false）
  downloads: Number,          // 下载次数（如果从社区导入）
  tags: [String],             // 标签（便于搜索和分类）

  createdBy: String,          // 创建者用户ID
  createdAt: Date,
  updatedAt: Date
}
```

**索引**：
- `_id`: 主键
- `category`: 普通索引
- `isEnabled`: 普通索引
- `tags`: 多值索引
- `author`: 普通索引

**系统预装类型**：

系统在首次安装时会预装以下示例类型，帮助用户快速上手：
- family_relative（亲戚关系）
- social_friend（好友关系）
- work_colleague（同事关系）
- education_classmate（同学关系）
- neighbor_community（邻里关系）

> 注：这些"预装类型"与用户自定义类型在技术上完全平等，都存储在 guanxi_types 集合中，用户可以修改或删除它们。

#### 1.1.5 guanxi（关系集合）
存储人物之间的关系。

```javascript
{
  _id: ObjectId,              // 关系ID
  userId: String,             // 所属用户ID

  // 关系双方
  fromCharacterId: ObjectId,  // 发起方人物ID
  toCharacterId: ObjectId,    // 接收方人物ID

  // 关系类型
  typeId: String,             // 关系类型ID
  typeName: String,           // 关系类型名称（冗余）

  // 关系强度
  strength: String,           // 关系强度：strong/medium/weak
  intimacy: Number,           // 亲密度（0-100）

  // 动态属性（根据关系类型不同而不同）
  attributes: {
    // 示例（亲戚关系）:
    // title: "父亲",
    // lineage: "直系",
    // generation: 1

    // 示例（好友关系）:
    // metAt: "大学",
    // commonHobbies: ["篮球", "游戏"],
    // friendLevel: "好友"
  },

  // 时间段（支持多个）
  periods: [{
    startTime: String,        // 开始时间 (YYYY-MM-DD)
    endTime: String,          // 结束时间 (YYYY-MM-DD)，null表示持续中
    status: String,           // 状态：active/ended/paused
    note: String,             // 时间段备注
    createdAt: Date
  }],

  // 关系描述
  description: String,        // 关系描述
  note: String,               // 备注
  tags: [String],             // 标签

  // 互动记录
  lastContactTime: Date,      // 最后联系时间
  contactFrequency: String,   // 联系频率：often/sometimes/rarely

  // 关系方向性
  isBidirectional: Boolean,   // 是否双向（默认true）
  reverseLabel: String,       // 反向标签（单向关系时使用）

  // 系统字段
  isActive: Boolean,          // 是否当前活跃（有未结束的时间段）
  createdAt: Date,
  updatedAt: Date,

  // 冗余字段（提升查询性能）
  fromCharacterName: String,
  toCharacterName: String,
  fromCharacterAvatar: String,
  toCharacterAvatar: String
}
```

**索引**：
- `userId`: 普通索引
- `{userId: 1, typeId: 1}`: 复合索引
- `{userId: 1, fromCharacterId: 1}`: 复合索引
- `{userId: 1, toCharacterId: 1}`: 复合索引
- `{fromCharacterId: 1, toCharacterId: 1}`: 复合索引（查找特定关系）
- `isActive`: 普通索引
- `lastContactTime`: 普通索引

**约束**：
- fromCharacterId 和 toCharacterId 不能相同（不能自己与自己建立关系）
- 同一对人物、同一类型的关系只能有一个
- periods 数组不能为空（至少有一个时间段）

#### 1.1.6 reminders（提醒集合）
存储提醒事项。

```javascript
{
  _id: ObjectId,
  userId: String,

  // 提醒对象
  targetType: String,         // 目标类型：character/guanxi/event
  targetId: ObjectId,         // 目标ID

  // 提醒信息
  type: String,               // 提醒类型：birthday/anniversary/contact/custom
  title: String,              // 提醒标题
  content: String,            // 提醒内容

  // 提醒时间
  remindTime: Date,           // 提醒时间
  isRecurring: Boolean,       // 是否重复
  recurringRule: String,      // 重复规则（如：yearly, monthly）

  // 状态
  status: String,             // 状态：pending/sent/dismissed
  sentAt: Date,               // 发送时间

  createdAt: Date,
  updatedAt: Date
}
```

**索引**：
- `{userId: 1, status: 1, remindTime: 1}`: 复合索引

#### 1.1.7 events（事件时间线集合）
存储关系中的重要事件。

```javascript
{
  _id: ObjectId,
  userId: String,

  // 关联对象
  guanxiId: ObjectId,         // 关联的关系ID
  characterIds: [ObjectId],   // 涉及的人物ID列表

  // 事件信息
  title: String,              // 事件标题
  description: String,        // 事件描述
  eventTime: String,          // 事件时间 (YYYY-MM-DD)
  eventType: String,          // 事件类型：meet/conflict/milestone/custom

  // 附件
  images: [String],           // 图片URL列表

  // 情感标记
  sentiment: String,          // 情感：positive/neutral/negative

  tags: [String],
  createdAt: Date,
  updatedAt: Date
}
```

**索引**：
- `{userId: 1, eventTime: -1}`: 复合索引（时间线查询）
- `guanxiId`: 普通索引
- `characterIds`: 普通索引

#### 1.1.8 graph_snapshots（图谱快照集合）
存储生成的关系图谱数据（缓存）。

```javascript
{
  _id: ObjectId,
  userId: String,

  // 图谱配置
  centerCharacterId: ObjectId, // 中心人物ID
  filters: {
    typeIds: [String],        // 筛选的关系类型
    timeRange: {              // 时间范围
      start: String,
      end: String
    },
    maxDepth: Number          // 最大深度
  },

  // 图谱数据
  graphData: {
    nodes: [{                 // 节点列表
      id: String,
      name: String,
      avatar: String,
      type: String,
      x: Number,              // 坐标（布局后）
      y: Number
    }],
    edges: [{                 // 边列表
      id: String,
      source: String,
      target: String,
      type: String,
      label: String
    }]
  },

  // 统计信息
  stats: {
    nodeCount: Number,
    edgeCount: Number,
    typeDistribution: {}      // 各类型关系数量
  },

  // 缓存信息
  expiresAt: Date,            // 过期时间
  createdAt: Date
}
```

**索引**：
- `{userId: 1, centerCharacterId: 1}`: 复合索引
- `expiresAt`: TTL索引（自动删除过期数据）

### 1.2 派生属性自动计算

某些属性可以从其他属性推算得出，系统应自动计算并填充，减少用户输入负担。

**生日、星座、生肖的推算关系**：

1. **阳历生日 → 星座**
   - 根据阳历生日的月份和日期确定星座
   - 星座规则（12星座及对应日期范围）：
     - 白羊座（Aries）: 03-21 ~ 04-19
     - 金牛座（Taurus）: 04-20 ~ 05-20
     - 双子座（Gemini）: 05-21 ~ 06-21
     - 巨蟹座（Cancer）: 06-22 ~ 07-22
     - 狮子座（Leo）: 07-23 ~ 08-22
     - 处女座（Virgo）: 08-23 ~ 09-22
     - 天秤座（Libra）: 09-23 ~ 10-23
     - 天蝎座（Scorpio）: 10-24 ~ 11-22
     - 射手座（Sagittarius）: 11-23 ~ 12-21
     - 摩羯座（Capricorn）: 12-22 ~ 01-19
     - 水瓶座（Aquarius）: 01-20 ~ 02-18
     - 双鱼座（Pisces）: 02-19 ~ 03-20

2. **阴历生日年份 → 生肖**
   - 根据农历年份确定生肖（12生肖轮回）
   - 生肖规则：
     - 鼠（Rat）、牛（Ox）、虎（Tiger）、兔（Rabbit）
     - 龙（Dragon）、蛇（Snake）、马（Horse）、羊（Goat）
     - 猴（Monkey）、鸡（Rooster）、狗（Dog）、猪（Pig）
   - 注意：农历年份以立春为界，不是阳历1月1日

3. **阴阳历互相转换**
   - 用户输入阳历生日 → 自动计算阴历生日
   - 用户输入阴历生日 → 自动计算阳历生日
   - 使用农历转换库（如：lunar-javascript、solarlunar等）
   - 转换范围：1900-2100年

**实现策略**：
- 用户创建/编辑人物时，只需填写一种生日（阳历或阴历）
- 系统自动：
  - 转换另一种生日格式
  - 根据阳历生日计算星座
  - 根据阴历年份计算生肖
- 所有计算结果存入 `immutableAttributes`
- 如果用户修改生日，相关属性自动重新计算

**特殊情况处理**：
- 阴历生日可能对应多个阳历日期（闰月）→ 提示用户选择
- 某些特殊年份的转换 → 使用专业农历库处理
- 虚构人物可能没有准确生日 → 允许不填写或手动填写星座/生肖

**数据存储**：
```javascript
// 存储示例
{
  birthday: {
    solar: "1990-05-15",      // 用户输入或自动计算
    lunar: "1990-04-21",      // 用户输入或自动计算
    primary: "solar"          // 用户指定的主要生日
  },
  immutableAttributes: {
    zodiacSign: "Taurus",     // 自动计算（金牛座）
    chineseZodiac: "Horse",   // 自动计算（马）
    bloodType: "A"            // 用户输入
  }
}
```
