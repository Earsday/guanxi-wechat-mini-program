## 🗂️ 文档使用指南

### 快速开始

**我是产品经理** → 阅读顺序：
1. 业务需求.md - 了解产品定位和功能
2. 关系类型设计目录 - 了解各类型的业务价值

**我是架构师** → 阅读顺序：
1. 业务需求.md - 理解业务需求
2. 技术概要设计.md - 把握整体架构
3. 关系类型设计规范.md - 理解类型系统架构
4. 技术详细设计各章节 - 深入实现细节

**我是开发工程师** → 阅读顺序：
1. 开发规范与标准.md - 了解开发规范
2. 技术概要设计.md - 理解系统架构
3. 技术详细设计对应章节 - 查看实现细节
4. 关系类型设计规范.md（如需开发新关系类型）

**我要开发新的关系类型** → 阅读顺序：
1. 关系类型设计/关系类型设计规范.md - 完整规范
2. 选择一个相似的预装类型文档作为参考
3. 技术详细设计-5-关系类型插件实现.md - 技术实现

## 📋 文档维护

### 更新文档时的注意事项

1. **业务需求变更** → 更新 `业务需求.md`，记录变更历史
2. **架构调整** → 更新 `技术概要设计.md`
3. **实现细节变更** → 更新对应的 `技术详细设计-X-XXX.md` 章节
4. **添加新关系类型** → 在 `关系类型设计/` 目录添加新文档，更新目录README
5. **跨文档引用** → 使用正确的相对路径：
   - 同目录文件：`./文件名.md`
   - 父目录文件：`../文件名.md`
   - 子目录文件：`./子目录/文件名.md`

### 文档一致性检查清单

- [ ] 版本号和日期一致
- [ ] 关系类型ID、名称、优先级一致
- [ ] 跨文档引用的配置值一致
- [ ] 内部链接有效（文件存在、锚点正确）
- [ ] 目录结构反映实际文件组织
