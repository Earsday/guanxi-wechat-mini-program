## 1. 非功能性需求

### 1.1 性能需求

**响应时间要求**：
- 小程序启动时间 < 3秒
- 人物列表加载（本地100条）< 0.5秒
- 关系图谱渲染（100个节点）< 2秒
- 本地搜索响应时间 < 0.3秒
- 切换中心人物重新生成图谱 < 2秒
- 数据导出（1000条数据）< 5秒
- 数据导入（1000条数据）< 10秒（含解密）
- 本地数据库查询响应 < 100ms

**资源占用要求**：
- 小程序包大小 < 4MB（主包 < 2MB）
- 内存占用 < 200MB
- 本地存储空间使用合理（提示用户清理）

**并发处理**：
- 支持多个异步操作同时进行
- 数据操作队列化，避免冲突

### 1.2 兼容性需求

**微信版本**：
- 支持微信版本 ≥ 7.0.0
- 基础库版本 ≥ 2.10.0

**操作系统**：
- iOS ≥ 10.0
- Android ≥ 5.0

**屏幕适配**：
- 支持不同屏幕尺寸（4.7寸至6.7寸）
- 支持不同屏幕分辨率（750rpx设计稿为基准）
- 支持横竖屏切换
- 支持 iPhone X 系列刘海屏适配

**系统特性**：
- 支持深色模式（Dark Mode）
- 支持系统字体大小设置

### 1.3 可用性需求

**操作简便性**：
- 所有关键操作不超过3步
- 提供快捷操作入口
- 常用功能一键可达

**用户引导**：
- 首次使用提供新手引导
- 关键功能提供操作提示
- 提供帮助文档和FAQ

**交互反馈**：
- 所有操作提供即时反馈
- 加载过程显示进度指示
- 错误提示清晰明确，提供解决建议
- 成功操作给予正向反馈

**容错机制**：
- 支持关键操作的撤销功能
- 删除等危险操作二次确认
- 输入验证和错误提示
- 异常情况优雅降级

**用户体验**：
- 界面简洁直观，信息层次清晰
- 符合微信设计规范
- 响应式设计，适配不同屏幕
- 动画流畅自然

### 1.4 扩展性需求

**架构扩展性**：
- 关系类型采用插件化架构，易于扩展
- 数据模型支持动态字段
- API 设计遵循 RESTful 规范
- 模块化设计，低耦合高内聚

**业务扩展性**：
- 支持多种关系网视角（不同中心人物）
- 支持新增关系类型
- 支持自定义属性字段
- 支持多个关系网项目管理

**国际化扩展**：
- 支持新增语言包
- 支持新增文化模板
- 姓名格式可扩展
- 性别选项可配置

**数据扩展性**：
- 支持大规模数据（10000+人物）
- 数据结构向前兼容
- 数据导入导出支持版本演进

### 1.5 国际化需求

#### 1.5.1 界面多语言

**支持语言**：
- 简体中文（zh-CN）
- 繁体中文（zh-TW）
- 英语（en-US）
- 日语（ja-JP）

**多语言内容**：
- 所有UI文本
- 提示信息
- 错误消息
- 帮助文档

**语言切换**：
- 根据系统语言自动切换
- 用户可手动选择界面语言
- 语言设置持久化
- 切换后实时生效

#### 1.5.2 人物属性国际化

**姓名格式支持**：

1. **中文姓名**（zh-CN）：
   - 字段：姓（lastName）+ 名（firstName）
   - 显示格式：姓+名（如：张三）
   - 示例：姓="张"，名="三" → 显示"张三"

2. **西方姓名**（en-US）：
   - 字段：FirstName + MiddleName（可选）+ LastName
   - 显示格式：FirstName MiddleName LastName
   - 简称格式：FirstName LastName
   - 示例：FirstName="John", MiddleName="Michael", LastName="Smith" → 显示"John Michael Smith"

3. **日语姓名**（ja-JP）：
   - 字段：姓（lastName）+ 名（firstName）
   - 显示格式：姓+名（如：山田太郎）
   - 支持假名和汉字

4. **其他文化姓名**：
   - 系统预留扩展接口
   - 可添加新的文化模板

**姓名处理规则**：
- 自动检测和解析姓名格式
- 根据文化背景提供对应的输入字段
- 生成统一的 displayName 用于显示
- 搜索时支持所有姓名字段

**性别选项国际化**：

**基础选项**（所有地区）：
- 男性（Male / 男性 / 男性）
- 女性（Female / 女性 / 女性）
- 其他（Other / 其他 / その他）
- 不愿透露（Prefer not to say / 不愿透露 / 無回答）

**扩展选项**（部分地区）：
- 非二元性别（Non-binary）
- 根据地区文化习惯显示

**文化背景字段**：
- culturalContext：标识人物所属文化区域（zh-CN、en-US、ja-JP等）
- 影响姓名格式、属性字段显示
- 支持混合文化背景的关系网

#### 1.5.3 日期时间国际化

**日期格式支持**：
- 中文：YYYY-MM-DD（2026-03-21）
- 美式：MM/DD/YYYY（03/21/2026）
- 欧式：DD/MM/YYYY（21/03/2026）
- 日式：YYYY年MM月DD日（2026年03月21日）

**日期处理**：
- 存储统一使用 ISO 8601 格式
- 显示根据用户偏好和地区自动格式化
- 日期选择器适配本地格式

**时区处理**：
- UTC时间存储
- 本地时区显示
- 时区自动转换

**本地化元素**：
- 节假日识别（不同地区）
- 生日提醒本地化
- 纪念日本地化

#### 1.5.4 数据导出国际化

**文件命名**：
- 格式：`guanxi_[英文名]_[本地语言名]_[UTC时间].gx[version]`
- 英文名：可选，便于国际搜索
- 本地语言名：必填
- UTC时间：统一使用UTC时区

**示例**：
- 中文环境：`guanxi_我的关系网_20260321T083045Z.gx0`
- 双语：`guanxi_hongloumeng_红楼梦_20260321T083045Z.gx0`
- 英文环境：`guanxi_MyNetwork_20260321T083045Z.gx0`

**数据内容**：
- 包含语言标识
- 包含文化背景信息
- 导入时自动适配

### 1.6 安全性需求

#### 1.6.1 数据隐私保护

**存储策略**：
- 默认本地存储，不上传云端
- 云端同步为可选功能
- 明确的隐私政策提示
- 用户完全控制数据位置

**数据分类**：
- 私密数据（个人真实关系）：优先本地存储
- 公开数据（虚构人物关系）：可选云端同步

#### 1.6.2 加密保护

**数据导出加密**：
- 算法：AES-256
- 密码要求：至少8位，包含字母和数字
- 密码强度提示
- 密码不存储在数据包中

**本地数据加密**：
- 敏感字段加密存储
- 密钥管理安全
- 定期密钥轮换

#### 1.6.3 访问控制

**应用锁**：
- 支持密码锁
- 支持指纹识别
- 支持面容识别
- 自动锁定时间可配置

**隐私模式**：
- 隐藏敏感信息显示
- 模糊化处理
- 快速切换隐私模式

#### 1.6.4 数据安全

**安全防护**：
- 防止 SQL 注入
- 防止 XSS 攻击
- 输入验证和过滤
- 安全的数据导入验证

**操作日志**：
- 记录关键操作（仅本地）
- 日志定期清理
- 日志加密存储

**数据备份**：
- 支持本地备份
- 备份文件加密
- 备份恢复验证

## 3. 技术规范

### 3.1 编码规范

**JavaScript 编码规范**：
- 遵循 ESLint 标准配置
- 使用 ES6+ 语法
- 异步操作使用 async/await
- 避免回调地狱

**代码风格**：
- 使用2个空格缩进
- 使用分号结束语句
- 字符串优先使用单引号
- 对象和数组使用尾随逗号

**代码质量**：
- 函数单一职责
- 避免深层嵌套（≤3层）
- 避免魔法数字，使用常量
- 及时处理错误，不使用空 catch

### 3.2 命名规范

**文件命名**：
- 组件：kebab-case（如：character-picker.js）
- 页面：kebab-case（如：character-list.js）
- 工具类：camelCase（如：dateUtil.js）

**变量命名**：
- 变量和函数：camelCase（如：userName）
- 常量：UPPER_SNAKE_CASE（如：MAX_DEPTH）
- 类名：PascalCase（如：GuanxiType）
- 私有属性：下划线前缀（如：_privateVar）

**语义化命名**：
- 使用有意义的名称
- 布尔值用 is/has/can 等前缀
- 事件处理函数用 on/handle 前缀
- 避免单字母变量（除循环索引）

### 3.3 注释规范

**文件头注释**：
```javascript
/**
 * 文件说明
 * @description 功能描述
 * @author 作者
 * @date 创建日期
 */
```

**函数注释**：
```javascript
/**
 * 函数说明
 * @param {Type} paramName - 参数说明
 * @return {Type} 返回值说明
 */
```

**关键逻辑注释**：
- 复杂算法添加注释
- 特殊处理说明原因
- TODO 标记待处理事项

### 3.4 版本控制规范

**分支管理**：
- main：主分支（稳定版本）
- develop：开发分支
- feature/*：功能分支
- bugfix/*：修复分支

**提交规范**：
- 使用语义化提交信息
- 格式：`type(scope): subject`
- 类型：feat/fix/docs/style/refactor/test/chore

**版本号规范**：
- 遵循语义化版本（Semantic Versioning）
- 格式：MAJOR.MINOR.PATCH
- 数据格式版本：.gxN 扩展名中的 N
